# EGOIST ACADEMY - Telegram Bot

Элегантный образовательный бот в стиле old money для обучения этикету, стилю, искусству, финансам и другим аспектам аристократической культуры.

## ✨ Возможности

- 📚 **12 образовательных модулей**: Этикет, Стиль, Искусство, Финансы, Путешествия, Право, Вино, Психология, Ораторство, Лидерство, Время и досуг, Политика
- 🤖 **AI-генерация уроков**: Персонализированные уроки через Groq API
- 📝 **Интерактивные тесты**: Проверка знаний после каждого урока
- 📊 **Система прогресса**: Отслеживание уровня и достижений
- ⭐ **Избранные уроки**: Сохранение интересных материалов
- 🎯 **Умные рекомендации**: Персональные предложения следующих тем
- 💬 **Вопросы к Felix**: AI-консьерж для дополнительных вопросов
- 🛡️ **Защита от спама**: Rate limiting и throttling middleware

## 🚀 Быстрый старт

### Автоматическая установка (Ubuntu)
```bash
sudo bash install.sh
```

### Ручная установка

1. **Установка зависимостей**
```bash
pip install -r requirements.txt
```

2. **Настройка**
```bash
cp .env.example .env
nano .env  # Заполните BOT_TOKEN и GROQ_API_KEY
```

3. **Запуск**
```bash
python main.py
```

## ⚙️ Конфигурация

Создайте файл `.env`:
```env
BOT_TOKEN=your_telegram_bot_token
GROQ_API_KEY=your_groq_api_key
GROQ_BASE_URL=https://api.groq.com/openai/v1
GROQ_MODEL=llama-3.3-70b-versatile
DATABASE_PATH=academy.db
LOG_FILE=bot.log
```

## 🔧 Технологии

- **Python 3.11+**
- **aiogram 3.4.1** - Telegram Bot API
- **SQLite** (aiosqlite) - База данных
- **Groq API** - AI генерация контента
- **FSM** - Управление состояниями диалогов

## ⚠️ Важные замечания

- **VPN/VPS**: Если api.telegram.org заблокирован, используйте VPN или разверните на VPS
- **API лимиты**: Groq API имеет лимиты запросов
- **Бэкапы**: Регулярно создавайте бэкапы БД командой `bash backup.sh`

## 🐛 Решение проблем

### Бот не отвечает
- Проверьте `BOT_TOKEN` в `.env`
- Убедитесь в подключении к интернету
- Проверьте логи в `bot.log`

### Ошибки генерации уроков
- Проверьте `GROQ_API_KEY`
- Убедитесь в лимитах API
- Проверьте логи

### Проблемы с БД
- Удалите `academy.db` и перезагрузите бота
- Проверьте права доступа к файлам
